# 車輛管理後台 MVP：Angular i18n + 調度 + 保養提醒

## 一、模組範圍界定

這個 MVP 對應到我們路線圖裡 **Phase 1** 的內部基礎工程（車行/平台方視角的後台，非旅客端）。先把三塊拆開看難易度：

| 子模組 | 難度 | 說明 |
|---|---|---|
| Angular i18n | ⭐⭐ | 框架內建功能，主要是工程配置與翻譯內容管理 |
| 車輛調度 | ⭐⭐⭐ | 需設計車輛狀態機、時間軸衝突偵測 |
| 保養紀錄與提醒 | ⭐⭐ | CRUD + 排程通知，邏輯單純 |

## 三、分階段開發順序

### 🟢 Step 1｜Angular i18n 基礎建置（1週）

用 Angular 內建 `@angular/localize`，這是最快能做完的部分。

```bash
ng add @angular/localize
```

```html
<!-- app.component.html -->
<h1 i18n="@@dashboard.title">車輛調度後台</h1>
<button i18n="@@dashboard.addVehicle">新增車輛</button>
```

```json
// angular.json — 設定多語系 build
"i18n": {
  "sourceLocale": "zh-TW",
  "locales": {
    "en": "src/locale/messages.en.xlf",
    "ja": "src/locale/messages.ja.xlf"
  }
}
```

**兩種路線二選一：**
- **Build-time i18n**（`@angular/localize`）：效能最好，但每個語系要獨立build，切換語言要重新導頁。適合語系少（中/英/日3個內）、SEO重要的場景。


因為後續會延伸到旅客端多語系（英/日/韓），**建議用 Transloco**，理由：翻譯檔可以用JSON存放、可對接CMS讓非工程人員更新翻譯字串、不需重新build。

```typescript
// transloco.config.ts
export const translocoConfig = {
  availableLangs: ['zh-TW', 'en', 'ja'],
  defaultLang: 'zh-TW',
  reRenderOnLangChange: true,
};
```

### 🟡 Step 2｜車輛調度模組（2-4週）

**核心邏輯：時間軸衝突偵測**，這是整個後台最需要設計思考的部分。

```typescript
// 調度核心邏輯：檢查車輛在指定時段是否可用
function isVehicleAvailable(
  vehicleId: string,
  requestedStart: Date,
  requestedEnd: Date,
  existingBookings: RentalBooking[]
): boolean {
  const conflicts = existingBookings.filter(booking =>
    booking.vehicleId === vehicleId &&
    booking.status !== 'cancelled' &&
    requestedStart < booking.endTime &&
    requestedEnd > booking.startTime
  );
  return conflicts.length === 0;
}
```

**UI 建議**：甘特圖式時間軸（可用 `angular-gantt` 或自製簡化版），車行人員一眼看到哪些車在哪個時段被占用，比純表單快很多。

**車輛狀態機**（避免調度邏輯散落各處）：
```
available → reserved（有預訂但未取車）→ rented（已取車）→ available（歸還）
available → maintenance（送修）→ available（修復）
```

任何狀態轉換都要寫成單一 service 方法，不要讓多個元件各自改 `status` 欄位，否則之後很難追蹤誰改的、為何改。

### 🟡 Step 3｜保養紀錄與提醒（1-2週）

**兩種提醒觸發規則**（通常兩者都要，取先到者）：
- 里程數：例如每 3000 km 提醒換機油
- 日期：例如每 6 個月做一次安全檢查

```typescript
function checkMaintenanceAlerts(
  vehicle: Vehicle,
  records: MaintenanceRecord[]
): MaintenanceAlert[] {
  const alerts: MaintenanceAlert[] = [];
  const lastRecord = records
    .filter(r => r.vehicleId === vehicle.id)
    .sort((a, b) => b.performedAt.getTime() - a.performedAt.getTime())[0];

  if (lastRecord?.nextDueMileage && vehicle.mileage >= lastRecord.nextDueMileage - 300) {
    alerts.push({
      vehicleId: vehicle.id,
      ruleType: 'mileage',
      threshold: lastRecord.nextDueMileage,
      status: vehicle.mileage >= lastRecord.nextDueMileage ? 'overdue' : 'upcoming',
    });
  }
  // 日期規則同理...
  return alerts;
}
```

**MVP 階段不需要**：自動化推播/簡訊通知（先用後台儀表板紅點提醒即可）、預測性保養AI（用不到，車輛數量不夠大）。

```bash
# 抽取翻譯字串

# 分別 build 三語系版本

ng build --configuration=en --output-path=dist/en
ng build --configuration=ja --output-path=dist/ja
```

**取捨提醒**：Build-time i18n 每個語系是獨立 bundle，切換語言 = 重新導頁（例如 `/en/dashboard` vs `/zh-TW/dashboard`）。這對**後台系統**完全沒問題（車行人員登入時選一次語言即可），比 Transloco 更輕量、無執行期翻譯開銷，也更符合你要「後台先行」的務實排序。

**部署時的路由切換**（Nginx範例）：
```nginx
location /en/ { alias /var/www/dist/en/; }
location /ja/ { alias /var/www/dist/ja/; }
location / { alias /var/www/dist/zh-TW/; }
```

## 四、車輛調度模組（Angular v22 Signals，OnPush 為預設）

Angular v22 元件不寫 `changeDetection` 就是 OnPush，天生適合用 Signals 管理調度狀態。

```typescript
// vehicle.model.ts
export interface Vehicle {
  id: string;
  plateNumber: string;
  type: 'scooter' | 'car';
  status: 'available' | 'rented' | 'maintenance' | 'reserved';
  mileage: number;
}

export interface RentalBooking {
  id: string;
  vehicleId: string;
  startTime: Date;
  endTime: Date;
  status: 'confirmed' | 'in_progress' | 'completed' | 'cancelled';
}
```

```typescript
// vehicle-dispatch.service.ts
import { Injectable, signal, computed } from '@angular/core';
import { Vehicle, RentalBooking } from './vehicle.model';

@Injectable({ providedIn: 'root' })
export class VehicleDispatchService {
  private vehicles = signal<Vehicle[]>([]);
  private bookings = signal<RentalBooking[]>([]);

  readonly availableVehicles = computed(() =>
    this.vehicles().filter(v => v.status === 'available')
  );

  isVehicleAvailable(vehicleId: string, start: Date, end: Date): boolean {
    return !this.bookings().some(b =>
      b.vehicleId === vehicleId &&
      b.status !== 'cancelled' &&
      start < b.endTime &&
      end > b.startTime
    );
  }

  updateVehicleStatus(vehicleId: string, status: Vehicle['status']): void {
    this.vehicles.update(list =>
      list.map(v => v.id === vehicleId ? { ...v, status } : v)
    );
  }
}
```

```typescript
// dispatch-board.component.ts
import { Component, inject } from '@angular/core';
import { VehicleDispatchService } from './vehicle-dispatch.service';

@Component({
  selector: 'app-dispatch-board',
  standalone: true,
  template: `
    <div class="grid grid-cols-1 md:grid-cols-3 gap-4 p-4">
      @for (vehicle of dispatchService.availableVehicles(); track vehicle.id) {
        <div class="rounded-lg border border-gray-200 p-4 shadow-sm">
          <p class="title-large">{{ vehicle.plateNumber }}</p>
          <span class="label-large text-green-600" i18n="@@vehicle.status.available">
            可租借
          </span>
        </div>
      }
    </div>
  `,
})
export class DispatchBoardComponent {
  dispatchService = inject(VehicleDispatchService);
}
```

## 五、保養提醒模組

```typescript
// maintenance.model.ts
export interface MaintenanceRecord {
  id: string;
  vehicleId: string;
  performedAt: Date;
  mileageAtService: number;
  nextDueMileage?: number;
  nextDueDate?: Date;
}

export interface MaintenanceAlert {
  vehicleId: string;
  ruleType: 'mileage' | 'date';
  status: 'upcoming' | 'overdue';
}
```

```typescript
// maintenance.service.ts
import { Injectable, signal, computed } from '@angular/core';
import { Vehicle } from './vehicle.model';
import { MaintenanceRecord, MaintenanceAlert } from './maintenance.model';

@Injectable({ providedIn: 'root' })
export class MaintenanceService {
  private records = signal<MaintenanceRecord[]>([]);
  private vehicles = signal<Vehicle[]>([]);

  readonly alerts = computed<MaintenanceAlert[]>(() => {
    const result: MaintenanceAlert[] = [];
    for (const vehicle of this.vehicles()) {
      const lastRecord = this.records()
        .filter(r => r.vehicleId === vehicle.id)
        .sort((a, b) => b.performedAt.getTime() - a.performedAt.getTime())[0];

      if (lastRecord?.nextDueMileage) {
        const isOverdue = vehicle.mileage >= lastRecord.nextDueMileage;
        const isUpcoming = vehicle.mileage >= lastRecord.nextDueMileage - 300;
        if (isOverdue || isUpcoming) {
          result.push({
            vehicleId: vehicle.id,
            ruleType: 'mileage',
            status: isOverdue ? 'overdue' : 'upcoming',
          });
        }
      }
    }
    return result;
  });
}
```

```html
<!-- maintenance-dashboard.component.html -->
<div class="p-4">
  <h2 class="headline-large" i18n="@@maintenance.title">保養提醒</h2>
  @for (alert of maintenanceService.alerts(); track alert.vehicleId) {
    <div
      class="rounded-md p-3 mb-2"
      [class.bg-red-50]="alert.status === 'overdue'"
      [class.bg-yellow-50]="alert.status === 'upcoming'"
    >
      @if (alert.status === 'overdue') {
        <span i18n="@@maintenance.overdue">已逾期，請盡快安排保養</span>
      } @else {
        <span i18n="@@maintenance.upcoming">即將到期</span>
      }
    </div>
  }
</div>
```
